# MeterSphere 用例格式转换工具

将 MeterSphere 测试用例从多步骤格式转换为合并格式，支持动态字段适配。

## 功能特性

- **多步骤合并**: 将同一用例的多个步骤行合并到一个单元格
- **智能编号处理**: 自动去除原有编号（如 `1、` `2.` `3）` 等），统一使用 `[编号]` 格式，避免重复编号
- **动态字段适配**: 自动检测源文件字段，只输出存在的字段
- **跨平台支持**: 支持 Windows 和 Linux 系统

## 转换说明

### 输入格式

源文件中，同一个用例的多个步骤分散在多行，通过 ID 字段判断用例边界：

| ID | 用例名称 | 步骤描述 | 预期结果 |
|----|---------|---------|---------|
| 1001 | 用例A | 步骤1 | 结果1 |
| | | 步骤2 | 结果2 |
| | | 步骤3 | 结果3 |
| 1002 | 用例B | 步骤1 | 结果1 |

### 输出格式

转换后，每个用例只占一行，步骤和预期结果合并到单个单元格：

| ID | 用例名称 | 步骤描述 | 预期结果 |
|----|---------|---------|---------|
| 1001 | 用例A | [1]步骤1<br>[2]步骤2<br>[3]步骤3 | [1]结果1<br>[2]结果2<br>[3]结果3 |
| 1002 | 用例B | [1]步骤1 | [1]结果1 |

### 编号处理

脚本会自动去除原有的编号格式：

| 原始内容 | 转换后 |
|---------|--------|
| `1、点击登录按钮` | `[1]点击登录按钮` |
| `2. 输入用户名` | `[2]输入用户名` |
| `3）点击确定` | `[3]点击确定` |

工具支持以下 MeterSphere 字段（按输出顺序）：

| 字段名 | 说明 |
|-------|------|
| ID | 用例ID |
| 用例名称 | 用例标题 |
| 所属模块 | 模块路径 |
| 前置条件 | 执行前条件 |
| 备注 | 备注信息 |
| 步骤描述 | 测试步骤（合并） |
| 预期结果 | 预期结果（合并） |
| 编辑模式 | STEP |
| 标签 | 用例标签 |
| 用例状态 | 状态信息 |
| 责任人 | 负责人 |
| 用例等级 | P0/P1/P2等 |
| 版本 | 版本号 |
| 验收用例 | 是否验收 |
| 测试方式 | 手动/自动 |
| 自动化实现 | 自动化状态 |
| 自动化类名 | 自动化类 |
| JIRA号 | 关联JIRA |
| 用例类型 | 功能/性能等 |
| 自动化说明 | 说明信息 |
| 评论 | 评论内容 |

## 常见问题

### Q: 转换后显示乱码？

A: 请确保源文件为 UTF-8 编码的 xlsx 格式。

### Q: 提示缺少必需字段？

A: 源文件必须包含以下字段：`ID`、`步骤描述`、`预期结果`。请检查表头是否正确。

### Q: 某些字段没有输出？

A: 工具只输出源文件中存在的字段。请检查源文件表头是否包含该字段名称。
